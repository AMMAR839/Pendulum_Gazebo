2. MODE KONTROL
------------------------------------------------------------

Mode yang dikirim ke GUI:
- 1 WAITING
- 2 HOMING
- 3 READY
- 4 SINE
- 5 FINISH
- 6 SWING_UP
- 7 BALANCE

Alur utama:
1. START dari GUI.
2. Tombol Y untuk homing.
3. Tombol X untuk swing-up.
4. Jika kondisi capture terpenuhi, bridge otomatis masuk BALANCE.
5. Tombol A hanya untuk memaksa BALANCE saat pendulum sudah dekat tegak.


3. HOMING
------------------------------------------------------------

Homing membawa cart ke tengah rail sebelum swing-up. Metodenya PD sederhana:

  if self.mode == MODE_HOMING:
      error = self.x_center_m - self.cart_x_m
      command = 2.8 * error - 0.35 * self.cart_v_mps
      if abs(error) < 0.006 and abs(self.cart_v_mps) < 0.03:
          self._set_mode_locked(MODE_READY)
          return 0.0, None
      return command, None

Makna:
- 2.8 * error menarik cart ke target tengah.
- -0.35 * cart_v_mps meredam supaya tidak overshoot.


4. SWING-UP BERBASIS ENERGI
------------------------------------------------------------

Swing-up dipakai untuk menaikkan energi pendulum sampai cukup dekat posisi atas.
Energi dihitung dari energi potensial dan kinetik:

  def _energy(self, theta_top):
      potential = self.pendulum_mass * 9.81 * self.pendulum_com * (
          1.0 + math.cos(theta_top)
      )
      kinetic = (
          0.5
          * self.pendulum_mass
          * self.pendulum_com
          * self.pendulum_com
          * self.pendulum_vel_radps
          * self.pendulum_vel_radps
      )
      return potential + kinetic

Target energi:

  target_energy = 2.0 * self.pendulum_mass * 9.81 * self.pendulum_com

Controller swing-up:

  phase = self.pendulum_vel_radps * math.cos(theta_top)
  energy_deficit = target_energy - energy

  force = -52.0 * self.swing_gain * energy_deficit * phase
  force -= 34.0 * self.swing_centering_gain * self.cart_x_m
  force -= 14.0 * self.swing_damping_gain * self.cart_v_mps

Makna:
- energy_deficit menentukan apakah pendulum masih butuh energi.
- phase menentukan arah dorongan cart.
- centering dan damping menjaga cart tidak keluar dari rail.

Saat pendulum masih diam di bawah, energi law belum punya arah. Karena itu
dipakai kick awal kecil:

  if abs(theta_deg) > 160.0 and abs(self.pendulum_vel_radps) < 0.60:
      if abs(self.cart_x_m) > 0.04:
          kick_direction = -math.copysign(1.0, self.cart_x_m)
      else:
          elapsed = time.monotonic() - self.swing_state_started
          kick_direction = 1.0 if int(elapsed / 0.45) % 2 == 0 else -1.0
      force += kick_direction * max(35.0, 70.0 * self.swing_kick_mps)


5. TRANSISI SWING_UP KE BALANCE
------------------------------------------------------------

Manual Book menjelaskan balance masuk ketika pendulum sudah dekat posisi atas,
sekitar -10 deg sampai 10 deg. Workspace ini mengikuti itu:

  self.declare_parameter("balance_capture_deg", 10.0)
  self.declare_parameter("balance_capture_rate_rad_s", 4.0)
  self.declare_parameter("balance_capture_cart_pos_m", 0.30)
  self.declare_parameter("balance_capture_cart_vel_mps", 3.0)

Syarat capture:

  return (
      abs(theta_top) < self.balance_capture
      and abs(self.pendulum_vel_radps) < self.balance_capture_rate
      and abs(self.cart_x_m) < capture_cart_pos
      and abs(self.cart_v_mps) < self.balance_capture_cart_vel
  )

Jika semua syarat terpenuhi, mode otomatis menjadi BALANCE:

  if self._ready_for_balance_locked(theta_top):
      self.x_center_m = clamp(
          self.cart_x_m,
          -self.rail_limit + 0.08,
          self.rail_limit - 0.08,
      )
      self._set_mode_locked(MODE_BALANCE)
      return self._gui_balance_command_locked(theta_top, dt), None

Kenapa tetap bisa susah:
- Pendulum sering lewat atas dengan theta_dot terlalu besar.
- Cart bisa terlalu jauh dari tengah.
- Motor model lebih lambat karena konstanta waktu 0.4 s.
- Tidak ada bantuan torsi engsel.


6. BALANCE FULL-STATE FEEDBACK
------------------------------------------------------------

Manual Book memakai bentuk:

  u = -(K_TH * e_theta + K_TH_D * e_theta_d
        + K_X * e_x + K_X_D * e_x_d)

Implementasi bridge memakai state yang sama, tetapi command akhirnya berupa
target kecepatan cart:

  command = (
      self.gains["K_TH"] * theta_top
      + self.gains["K_TH_D"] * self.pendulum_vel_radps
      - self.gains["K_X"] * x_error_m
      - self.gains["K_X_D"] * self.cart_v_mps
      - self.gains["K_X_INT"] * self.x_integral_cm_s
  )

State yang dipakai:
- theta_top: sudut terhadap posisi atas, 0 deg berarti tegak.
- pendulum_vel_radps: kecepatan sudut.
- x_error_m: posisi cart terhadap titik tengah balance.
- cart_v_mps: kecepatan cart.
- x_integral_cm_s: integral error posisi cart.

Jika sudut keluar terlalu jauh, balance dianggap gagal dan otomatis kembali ke
SWING_UP:

  def _lost_balance_locked(self, theta_top):
      return (
          abs(theta_top) > self.balance_fallback
          or abs(self.cart_x_m) > (self.rail_limit + 0.02)
      )

Default fallback:

  self.declare_parameter("balance_fallback_deg", 50.0)


7. MODEL MOTOR ALA MANUAL BOOK
------------------------------------------------------------

Manual Book menunjukkan hasil sweep motor:
- p1 = 189.1
- p2 = 3212
- time constant sekitar 0.4 s

Di bridge, command controller diubah menjadi PWM ekuivalen:

  self.motor_pwm = (
      self.motor_pwm_deadband
      + self.motor_pwm_per_cmps * abs(command_cmps)
  )

Lalu motor diberi respons orde satu:

  alpha = 1.0 - math.exp(-dt / max(self.motor_time_constant_s, 1e-3))
  self.motor_target_mps = target_mps
  self.motor_velocity_mps += alpha * (target_mps - self.motor_velocity_mps)

Velocity servo mengubah target motor menjadi gaya cart:

  velocity_error = self.motor_velocity_mps - self.cart_v_mps
  velocity_error_dot = (
      velocity_error - self.prev_motor_velocity_error
  ) / max(dt, 1e-3)

  effort = (
      self.motor_velocity_servo_p * velocity_error
      + self.motor_velocity_servo_d * velocity_error_dot
  )

Parameter motor:

  self.declare_parameter("motor_pwm_deadband", 3212.0)
  self.declare_parameter("motor_pwm_per_cmps", 189.1)
  self.declare_parameter("motor_time_constant_s", 0.40)
  self.declare_parameter("motor_velocity_servo_p", 90.0)
  self.declare_parameter("motor_velocity_servo_d", 4.0)
  self.declare_parameter("effort_limit_n", 60.0)


8. PERBEDAAN DENGAN WORKSPACE LAMA
------------------------------------------------------------

ros2_pendulum_ws:
- Dibuat supaya lebih mudah demo tegak.
- Bisa memakai gaya cart besar.
- Bisa memakai bantuan torsi kecil di engsel pendulum.
- Cocok untuk visualisasi bahwa swing-up dan balance bisa terjadi.

pendulum_real_ws:
- Lebih dekat Manual Book.
- Cart force dan travel tetap mengikuti dimensi manual.
- Motor punya deadband dan delay.
- Capture balance di sekitar posisi tegak.
- Memakai assist torsi kecil di pendulum_hinge saat catch/balance supaya
  simulasi bisa berdiri stabil.
